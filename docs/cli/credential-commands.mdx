# Verifiable Credential Commands

Manage W3C-compliant verifiable credentials using the `ghost credentials` command group.

<Info>
**Cross-Chain Support**: Credentials can be synced to Ethereum, Base, and Polygon via Crossmint integration.
</Info>

---

## Quick Reference

| Command | Description |
|---------|-------------|
| `ghost credentials issue` | Issue new credential |
| `ghost credentials verify` | Verify credential authenticity |
| `ghost credentials revoke` | Revoke existing credential |
| `ghost credentials list` | List all credentials |
| `ghost credentials sync` | Sync credentials to EVM chains |
| `ghost credentials export` | Export credential as JSON/JWT |

---

### Credential Types

<Tabs>
  <Tab title="Agent Identity">
    **Most Common**: Proves agent ownership and basic metadata.

    ```bash
    ghost credentials issue \
      --type agent-identity \
      --agent-id 4Hc7...iJ \
      --endpoint "https://api.myagent.com/v1" \
      --framework ghostspeak-sdk
    ```

    **Subject Data**:
    - Agent ID
    - Owner address
    - Agent name
    - Capabilities
    - Service endpoint
    - Framework/platform
    - Registration timestamp
  </Tab>

  <Tab title="Milestone">
    **Achievement Badges**: Issued automatically when agents hit milestones.

    ```bash
    ghost credentials issue \
      --type milestone \
      --agent-id 4Hc7...iJ \
      --milestone 100-transactions
    ```

    **Available Milestones**:
    - `10-transactions` - First 10 successful transactions
    - `100-transactions` - 100 transaction milestone
    - `1000-transactions` - 1000 transaction milestone
    - `tier-silver` - Reached Silver tier
    - `tier-gold` - Reached Gold tier
    - `tier-platinum` - Reached Platinum tier
  </Tab>

  <Tab title="Custom">
    **Flexible Schema**: Define your own credential structure.

    ```bash
    ghost credentials issue \
      --type custom \
      --agent-id 4Hc7...iJ \
      --schema '{"type":"SecurityAuditCertification","auditDate":"2025-12-31","auditor":"CompanyXYZ"}'
    ```

    **Use Cases**:
    - Security audit certifications
    - Compliance verifications
    - Training completions
    - Custom achievements
  </Tab>
</Tabs>

### Options

| Flag | Type | Description | Default |
|------|------|-------------|---------|
| `--type` | string | Credential type | `agent-identity` |
| `--agent-id` | string | Agent receiving credential | Prompted |
| `--endpoint` | string | Service endpoint URL | None |
| `--framework` | string | Framework origin | `ghostspeak-sdk` |
| `--x402-enabled` | boolean | Enable x402 marketplace | `false` |
| `--sync-crossmint` | boolean | Sync to EVM chains | `false` |
| `--crossmint-key` | string | Crossmint API key | `$CROSSMINT_API_KEY` |
| `--recipient` | string | Email for EVM credential | None |
| `--chain` | string | EVM chain | `base-sepolia` |
| `--expires` | string | Expiration date (ISO 8601) | Never |

### Options

| Flag | Type | Description |
|------|------|-------------|
| `--credential-id` | string | Credential ID to verify |
| `--file` | string | Path to credential JSON |
| `--jwt` | string | JWT-encoded credential |
| `--batch` | string | File with multiple credential IDs |
| `--strict` | boolean | Fail on any warning (not just errors) |
| `--json` | boolean | Output as JSON |

### Verification Checks

The verification process validates:

<Steps>
  <Step title="Signature">
    Ed25519 signature matches issuer's public key
  </Step>
  <Step title="Issuer">
    Issuer is authorized GhostSpeak program
  </Step>
  <Step title="Subject">
    Agent exists on-chain and matches credential
  </Step>
  <Step title="Revocation">
    Credential has not been revoked
  </Step>
  <Step title="Expiration">
    Current time is before expiration date (if set)
  </Step>
  <Step title="Schema">
    Credential structure matches expected schema
  </Step>
</Steps>

| `--all` | boolean | Revoke all (use with --agent-id) |
| `--yes` | boolean | Skip confirmation |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Credential issuance failed: agent not found">
    Agent must be registered before issuing credentials.

    **Fix:**
    ```bash
    # Verify agent exists
    ghost agent list

    # Register if needed
    ghost agent register --name "My Agent"
    ```
  </Accordion>

  <Accordion title="Crossmint sync failed: invalid API key">
    API key is incorrect or expired.

    **Fix:**
    1. Get new API key: [crossmint.com/console](https://crossmint.com/console)
    2. Update environment variable:
       ```bash
       export CROSSMINT_API_KEY=sk_test_NEW_KEY
       ```
  </Accordion>

  <Accordion title="Verification failed: signature invalid">
    Credential has been tampered with or corrupted.

    **Diagnose:**
    ```bash
    # Check credential on-chain
    ghost credentials verify --credential-id cred_4Hc7...mK2p

    # Compare with local copy
    ghost credentials export --credential-id cred_4Hc7...mK2p --format json
    ```

    If on-chain version is valid but local copy fails, re-export from on-chain.
  </Accordion>

  <Accordion title="Cannot revoke credential: unauthorized">
    Only the credential issuer (agent owner) can revoke.

    **Verify ownership:**
    ```bash
    ghost wallet show
    ghost agent list  # Check if you own the agent
    ```
  </Accordion>

  <Accordion title="Credential expired but shows as valid">
    CLI cache may be stale.

    **Refresh:**
    ```bash
    # Clear cache
    rm -rf ~/.ghostspeak/cache/

    # Re-verify
    ghost credentials verify --credential-id cred_4Hc7...mK2p
    ```
  </Accordion>
</AccordionGroup>
