## Overview

The `ghostspeak credentials` command group lets you manage your Agent Identity and sync it across blockchains (Solana to EVM) using W3C Verifiable Credentials. This is essential for cross-chain reputation and participating in opportunities like the **IPO Whitelist**.

## Commands

### ghostspeak credentials sync

Sync your Solana Agent Identity to an EVM-compatible verifiable credential via Crossmint.

**This command:**

1. Reads your local keypair
2. Prompts for your Agent ID (if not provided)
3. Generates a "Subject Data" payload with your agent's public key and metadata
4. Signs this payload with your Solana wallet
5. Submits it to the Crossmint API to issue a VC on the Base network

**Usage:**

```bash
ghostspeak credentials sync
```

**Options:**

| Option                | Description                                          |
| --------------------- | ---------------------------------------------------- |
| `-a, --agent <ID>`    | The Solana public key of the Agent to sync           |
| `-k, --key <KEY>`     | Crossmint API Key (overrides env var)                |
| `-e, --email <EMAIL>` | Email address to associate with the credential       |
| `-n, --network <NET>` | Solana RPC network (devnet/mainnet). Default: devnet |

## Credential Types

| Type              | Description                                  | Use Case                           |
| ----------------- | -------------------------------------------- | ---------------------------------- |
| **AgentIdentity** | Proves you own a registered GhostSpeak agent | IPO whitelists, cross-chain access |
| **Reputation**    | Proves your on-chain reputation score        | Trust signals in other ecosystems  |
| **JobCompletion** | Proves completed work history                | Portfolio and credibility          |

## Environment Variables

Set these for non-interactive usage:

```bash
# Required for sync
CROSSMINT_API_KEY=your_api_key

# Optional
CROSSMINT_API_URL=https://www.crossmint.com/api
CROSSMINT_AGENTIDENTITY_TEMPLATE_ID=tmpl_xxx
```
