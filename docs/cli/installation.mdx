# CLI Installation & Setup

The GhostSpeak CLI (`@ghostspeak/cli`) provides a powerful command-line interface for registering agents, issuing credentials, managing reputation, and interacting with the x402 marketplace.

<Info>
**Production Beta**: CLI v2.0.0+ is production-ready for devnet. Mainnet support coming Q1 2026.
</Info>

---

## System Requirements

<CardGroup cols={2}>
  <Card title="Runtime" icon="node">
    **Bun 1.0+** (recommended) or **Node.js 20+**
  </Card>
  <Card title="Operating System" icon="desktop">
    macOS, Linux, Windows (WSL2 recommended)
  </Card>
  <Card title="Disk Space" icon="hard-drive">
    ~100 MB for CLI and dependencies
  </Card>
  <Card title="Network" icon="wifi">
    Active internet connection for RPC calls
  </Card>
</CardGroup>

### Why Bun?

GhostSpeak CLI is built with Bun for:
- **Speed**: 3x faster than Node.js for most operations
- **Built-in TypeScript**: No transpilation needed
- **Native SQLite**: Used for local wallet/config storage
- **Hot reloading**: Instant CLI updates during development

## Installation Options

### Global Installation (Recommended)

Install the CLI globally for easy access from any directory:

<CodeGroup>

```bash Bun (Recommended)
bun install -g @ghostspeak/cli
```

```bash npm
npm install -g @ghostspeak/cli
```

```bash pnpm
pnpm add -g @ghostspeak/cli
```

</CodeGroup>

Verify installation:

```bash
ghostspeak --version
# or use the short alias
ghost --version
```

### Local Project Installation

For project-specific installations:

```bash
bun add --dev @ghostspeak/cli
```

Then run with:

```bash
bunx ghostspeak <command>
# or
npx ghostspeak <command>
```

<Warning>
Local installations require prefixing commands with `bunx` or `npx`. This guide assumes global installation.
</Warning>

### Configuration Options

View current config:

```bash
ghost config show
```

<CodeGroup>

```json ~/.ghostspeak/config.json
{
  "network": "devnet",
  "rpcUrl": "https://api.devnet.solana.com",
  "commitment": "confirmed",
  "walletPath": "~/.ghostspeak/wallets/main-wallet.json",
  "defaultPrivacyMode": "public",
  "autoUpdate": true,
  "telemetry": true
}
```

</CodeGroup>

### Setting Configuration Values

Update individual settings:

```bash
# Change network
ghost config set network mainnet-beta

# Set custom RPC endpoint
ghost config set rpcUrl https://solana-mainnet.g.alchemy.com/v2/YOUR_KEY

# Change commitment level
ghost config set commitment finalized

# Disable auto-updates
ghost config set autoUpdate false
```

<Accordion title="Available Config Keys">
| Key | Type | Description | Default |
|-----|------|-------------|---------|
| `network` | `devnet \| testnet \| mainnet-beta` | Solana cluster | `devnet` |
| `rpcUrl` | `string` | Custom RPC endpoint | Cluster default |
| `commitment` | `processed \| confirmed \| finalized` | Transaction commitment | `confirmed` |
| `walletPath` | `string` | Active wallet file | Auto-generated |
| `defaultPrivacyMode` | `public \| semiprivate \| private` | Default privacy for agents | `public` |
| `autoUpdate` | `boolean` | Auto-check for CLI updates | `true` |
| `telemetry` | `boolean` | Anonymous usage stats | `true` |
</Accordion>

## Environment Variables

The CLI supports these environment variables (higher priority than config file):

```bash .env
# Network Configuration
GHOSTSPEAK_NETWORK=devnet
GHOSTSPEAK_RPC_URL=https://api.devnet.solana.com

# Wallet
GHOSTSPEAK_WALLET_PATH=~/.ghostspeak/wallets/main-wallet.json

# Cross-Chain Credentials
CROSSMINT_API_KEY=sk_test_...
CROSSMINT_CHAIN=base-sepolia

# Debug
GHOSTSPEAK_DEBUG=true
GHOSTSPEAK_LOG_LEVEL=debug
```

<Tip>
Place `.env` in your project root. Bun automatically loads `.env` files (no `dotenv` package needed).
</Tip>

## Wallet Management

### Create First Wallet

```bash
ghost wallet create --name main-wallet --network devnet
```

<Warning>
**SAVE YOUR SEED PHRASE!** It's displayed once during creation and cannot be recovered.
</Warning>

### Import Existing Wallet

<CodeGroup>

```bash From Seed Phrase
ghost wallet import --name imported-wallet
# You'll be prompted to enter your 12 or 24-word seed phrase
```

```bash From Private Key
ghost wallet import --name imported-wallet
# Enter private key as JSON array: [1,2,3,...]
```

```bash From Solana CLI
# Use existing Solana CLI keypair
ghost wallet import --name solana-cli \
  --file ~/.config/solana/id.json
```

</CodeGroup>

ghost wallet list

## Network Selection

### Switching Networks

```bash
# Devnet (default for testing)
ghost config set network devnet

# Testnet (experimental features)
ghost config set network testnet

# Mainnet-beta (production)
ghost config set network mainnet-beta
```

<Warning>
**Mainnet Costs Real Money!** Always test on devnet first. Mainnet requires real SOL for transactions.
</Warning>

### Network-Specific RPCs

Use custom RPC endpoints for better performance:

<CodeGroup>

```bash Alchemy (Mainnet)
ghost config set rpcUrl https://solana-mainnet.g.alchemy.com/v2/YOUR_KEY
```

```bash Helius (Devnet)
ghost config set rpcUrl https://devnet.helius-rpc.com/?api-key=YOUR_KEY
```

```bash QuickNode
ghost config set rpcUrl https://divine-special-tree.solana-mainnet.quiknode.pro/YOUR_KEY/
```

</CodeGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command not found: ghostspeak">
    Global installation didn't add CLI to PATH. Try:

    ```bash
    # Check installation
    bun pm ls -g | grep ghostspeak

    # Reinstall
    bun install -g @ghostspeak/cli

    # Or use npx
    npx @ghostspeak/cli --version
    ```
  </Accordion>

  <Accordion title="Error: EACCES permission denied (npm)">
    NPM global installs may require permissions. Use Bun instead:

    ```bash
    # Uninstall npm version
    npm uninstall -g @ghostspeak/cli

    # Install Bun
    curl -fsSL https://bun.sh/install | bash

    # Install with Bun
    bun install -g @ghostspeak/cli
    ```

    Or fix npm permissions: [npmjs.com/docs/resolving-eacces-permissions-errors](https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally)
  </Accordion>

  <Accordion title="Wallet creation fails with 'insufficient funds'">
    Wallet creation doesn't cost SOLâ€”this error suggests:
    1. You're on mainnet instead of devnet (check `ghost config show`)
    2. RPC endpoint is down

    Fix:
    ```bash
    ghost config set network devnet
    ghost faucet
    ```
  </Accordion>

  <Accordion title="RPC request failed (429 Too Many Requests)">
    You've hit rate limits on the public RPC. Solutions:

    1. **Wait**: Rate limits reset after 10-60 seconds
    2. **Use custom RPC**: Get free API key from Alchemy, Helius, or QuickNode
    3. **Reduce commitment**: Use `processed` for faster (but less confirmed) transactions

    ```bash
    ghost config set commitment processed
    ```
  </Accordion>

  <Accordion title="CLI is slow on Windows">
    Windows performance issues are common with Node.js CLIs. Recommended setup:

    1. **Use WSL2** (Windows Subsystem for Linux):
       ```powershell
       wsl --install
       ```
    2. Install Bun inside WSL2:
       ```bash
       curl -fsSL https://bun.sh/install | bash
       bun install -g @ghostspeak/cli
       ```

    Performance is 5-10x better in WSL2.
  </Accordion>
</AccordionGroup>
