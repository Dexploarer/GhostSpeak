# Staking Commands

Manage GHOST token staking using the `ghost staking` command group (alias: `ghost stake`).

<Info>
**GHOST Token**: Native utility token for GhostSpeak protocol. Stake for unlimited verifications, revenue share, and governance rights.
</Info>

---

## Quick Reference

| Command | Shorthand | Description |
|---------|-----------|-------------|
| `ghost staking stake` | `ghost stake s` | Stake GHOST tokens |
| `ghost staking unstake` | `ghost stake u` | Unstake tokens (14-day unbonding) |
| `ghost staking claim` | `ghost stake c` | Claim staking rewards |
| `ghost staking status` | `ghost stake st` | View staking status |
| `ghost staking rewards` | `ghost stake r` | Check pending rewards |
| `ghost staking compound` | - | Auto-compound rewards |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Insufficient GHOST balance to stake">
    You need GHOST tokens to stake.

    **On Devnet**:
    ```bash
    ghost airdrop --amount 1000
    ```

    **On Mainnet**:
    1. Buy GHOST on exchange (Raydium, Orca, etc.)
    2. Transfer to your CLI wallet
    3. Verify balance: `ghost wallet balance`
  </Accordion>

  <Accordion title="Staking transaction failed">
    Common causes:
    - Insufficient SOL for transaction fees
    - Amount below minimum tier (100 GHOST)
    - Network congestion

    **Fix**:
    ```bash
    # Check SOL balance
    ghost wallet balance

    # Get SOL (devnet)
    ghost faucet

    # Retry with lower amount
    ghost staking stake --amount 100
    ```
  </Accordion>

  <Accordion title="Rewards not accumulating">
    Rewards accrue daily starting 24 hours after staking.

    **Check**:
    ```bash
    # View staking start time
    ghost staking status

    # If >24 hours, force refresh
    ghost staking rewards --refresh
    ```

    **If still 0 after 48 hours**:
    ```bash
    ghost config diagnose
    ```
  </Accordion>

  <Accordion title="Cannot unstake - insufficient available">
    You can't unstake more than your staked amount minus pending unbonding.

    **Check status**:
    ```bash
    ghost staking status

    # View unbonding
    ghost staking status --unbonding
    ```

    **Example**:
    - Staked: 500 GHOST
    - Unbonding: 200 GHOST
    - Available to unstake: 300 GHOST
  </Accordion>

  <Accordion title="Lost tier benefits after unstaking">
    **Expected**: Tier benefits are immediately lost when stake drops below threshold.

    **Example**:
    - Had 500 GHOST (Silver tier)
    - Unstaked 250 GHOST
    - Now 250 GHOST (Bronze tier)
    - Silver benefits (unlimited verifications, 8% APY) â†’ Lost

    **To restore**:
    ```bash
    # Cancel unstaking if within 14 days
    ghost staking cancel-unstake

    # Or stake more
    ghost staking stake --amount 250
    ```
  </Accordion>

  <Accordion title="Revenue share is 0">
    Revenue share depends on protocol fee volume.

    **Low Volume (Devnet/Early Mainnet)**:
    - Few transactions = low fees = low revenue share
    - Revenue share may be &lt;0.01 GHOST initially

    **Check volume**:
    ```bash
    ghost staking rewards --revenue-details
    ```

    **Revenue share grows with**:
    - More agents using protocol
    - Higher transaction volume
    - More x402 marketplace activity
  </Accordion>
</AccordionGroup>

## Staking Economics

<Accordion title="Where Do Staking Rewards Come From?">
Staking rewards are funded by:

1. **Inflation** (70%):
   - 3% annual token inflation
   - Distributed to stakers proportionally
   - Decreases over time (halving every 4 years)

2. **Protocol Fees** (30%):
   - Transaction fees (1-2% per transaction)
   - x402 marketplace fees (2.5%)
   - Credential issuance fees
   - 30% of fees go to stakers as revenue share
