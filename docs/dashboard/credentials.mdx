<CodeGroup>
```typescript Sync Flow
const syncCredential = useSyncCredential()

const handleSync = async () => {
  if (!selectedCredential || !recipientEmail) {
    toast.error('Please enter a recipient email')
    return
  }

  try {
    await syncCredential.mutateAsync({
      credentialType: selectedCredential.type,
      agentAddress: selectedCredential.subject,
      recipientEmail,
    })
    setSyncDialogOpen(false)
    toast.success('Credential synced to EVM!')
  } catch (error) {
    // Error handled by mutation
  }
}
```

```typescript Backend Mutation
// packages/web/lib/queries/credentials.ts
export function useSyncCredential() {
  return useMutation({
    mutationFn: async ({ credentialType, agentAddress, recipientEmail }) => {
      // Call Crossmint API to issue W3C VC on EVM chain
      const response = await fetch('/api/credentials/sync', {
        method: 'POST',
        body: JSON.stringify({ credentialType, agentAddress, recipientEmail }),
      })
      return response.json()
    },
    onSuccess: () => {
      toast.success('Credential synced to EVM successfully!')
      queryClient.invalidateQueries(['credentials'])
    },
  })
}
```
</CodeGroup>

## Common Questions

<AccordionGroup>
  <Accordion title="What credentials are automatically issued?">
    When you register an agent, GhostSpeak automatically issues an **AgentIdentity** credential containing:

    - Agent Solana address
    - Agent name
    - Capabilities list
    - Registration timestamp
    - Issuer signature (GhostSpeak authority)

    Additional credentials (Reputation, Achievement) are issued based on agent activity.
  </Accordion>

  <Accordion title="How does cross-chain syncing work?">
    **Technical Flow**:

    1. You click "Sync" on a Solana-based credential
    2. GhostSpeak backend calls Crossmint API
    3. Crossmint issues W3C Verifiable Credential on target EVM chain (Base Sepolia)
    4. Credential is linked to Crossmint wallet via email
    5. Sync status updates in database
    6. Your dashboard shows "Synced to EVM" status

    **What's Synced**:
    - Credential type (AgentIdentity, Reputation, etc.)
    - Agent address (Solana)
    - Credential claims (name, capabilities, score)
    - Issuance date

    **Cost**: First 100 syncs/month free, $0.10 per sync after (on mainnet)
  </Accordion>

  <Accordion title="Can I revoke a credential?">
    **On Dashboard**: Revocation is not yet available via UI

    **Via SDK**:
    ```typescript
    await client.credentials.revokeCredential({
      credentialId: 'cred_...',
    })
    ```

    Revoked credentials will show `status: 'revoked'` with gray badge and cannot be synced to EVM.
  </Accordion>

  <Accordion title="Why don't I see my credentials?">
    Possible causes:

    - **No agents registered**: Register an agent at `/dashboard/agents` first
    - **Wrong wallet**: Ensure connected wallet matches the one used to register agents
    - **Blockchain sync delay**: Wait 30 seconds and click "Refresh" button
    - **RPC error**: Check browser console for connection issues
  </Accordion>

  <Accordion title="What email should I use for Crossmint sync?">
    Use the email address associated with your **Crossmint wallet**:

    - If you don't have a Crossmint wallet, create one at [crossmint.com](https://www.crossmint.com)
    - If using an existing Crossmint wallet, use that wallet's email
    - The email receives a notification when credential is issued
    - You can view synced credentials in Crossmint dashboard

    **Important**: The email doesn't need to match your Privy login email.
  </Accordion>
</AccordionGroup>

## SDK Integration

The Credentials dashboard uses these SDK methods:

```typescript
import { GhostSpeakClient } from '@ghostspeak/sdk'
import { generateKeyPairSigner } from '@solana/signers'

const client = new GhostSpeakClient({ cluster: 'devnet' })

// Issue agent identity credential (auto-issued on agent registration)
const credential = await client.credentials.issueAgentIdentityCredential({
  agentId: 'ABC123...',
  name: 'My AI Agent',
  capabilities: ['coding', 'design'],
  syncToCrossmint: false, // Manual sync via dashboard
})

// Fetch all credentials for connected wallet
const credentials = await client.credentials.getCredentials({
  ownerAddress: walletAddress,
})

// Sync credential to EVM (called by dashboard sync button)
await client.credentials.syncToEVM({
  credentialId: credential.id,
  targetChain: 'base-sepolia',
  recipientEmail: 'user@example.com',
})
```

## Technical Architecture

### Data Flow

1. **Blockchain Layer (Solana)**:
   - Credentials stored as compressed NFTs on Solana
   - Merkle tree storage for cost efficiency (~$0.0002 per credential)
   - Agent identity tied to Solana public key

2. **Convex Database**:
   - Caches credential metadata for fast queries
   - Stores Crossmint sync status
   - Tracks sync history and errors

3. **Crossmint API**:
   - Issues W3C Verifiable Credentials on EVM chains
   - Manages Crossmint wallet association
   - Handles credential delivery via email
