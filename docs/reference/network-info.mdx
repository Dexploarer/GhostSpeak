### SDK RPC Configuration

```typescript
import { GhostSpeakClient } from '@ghostspeak/sdk'

// Option 1: Use default public RPC
const client = new GhostSpeakClient({
  cluster: 'devnet',
})

// Option 2: Custom RPC endpoint
const client = new GhostSpeakClient({
  cluster: 'devnet',
  rpcUrl: 'https://devnet.helius-rpc.com/?api-key=YOUR_KEY',
})

// Option 3: Advanced configuration
const client = new GhostSpeakClient({
  cluster: 'devnet',
  rpcUrl: 'https://api.devnet.solana.com',
  rpcConfig: {
    commitment: 'confirmed',
    confirmTransactionInitialTimeout: 60000, // 60 seconds
  },
})
```

## Commitment Levels

Choose the right commitment level for your use case:

<Tabs>
  <Tab title="Processed">
    **Fastest (not recommended for production)**

    ```typescript
    { commitment: 'processed' }
    ```

    - Transaction validated by leader
    - Not guaranteed to be finalized
    - Latency: ~400ms
    - Use for: Immediate UI feedback
  </Tab>

  <Tab title="Confirmed">
    **Recommended for most operations**

    ```typescript
    { commitment: 'confirmed' }
    ```

    - Supermajority of cluster confirmed
    - Low rollback risk (~0.01%)
    - Latency: ~1-2 seconds
    - Use for: Standard transactions
  </Tab>

  <Tab title="Finalized">
    **Safest (use for high-value operations)**

    ```typescript
    { commitment: 'finalized' }
    ```

    - Cluster finality achieved
    - Zero rollback risk
    - Latency: ~13 seconds (32 slots)
    - Use for: Credential issuance, staking
  </Tab>
</Tabs>

<Warning>
**GhostSpeak Default**: We use `confirmed` commitment for best balance of speed and safety.
</Warning>

## WebSocket Endpoints

For real-time updates (account changes, transaction confirmations):

```typescript
import { createSolanaRpc, createSolanaRpcSubscriptions } from '@solana/rpc'

// HTTP RPC
const rpc = createSolanaRpc('https://api.devnet.solana.com')

// WebSocket RPC
const rpcSubscriptions = createSolanaRpcSubscriptions('wss://api.devnet.solana.com')

// Subscribe to account changes
const subscription = await rpcSubscriptions
  .accountNotifications(agentAddress, { commitment: 'confirmed' })
  .subscribe(async (notification) => {
    console.log('Agent account updated:', notification)
  })
```

**WebSocket Endpoints:**

- **Devnet**: `wss://api.devnet.solana.com`
- **Mainnet**: `wss://api.mainnet-beta.solana.com`

## Rate Limiting Best Practices

<AccordionGroup>
  <Accordion title="Implement Exponential Backoff">
    ```typescript
    async function retryRpcCall<T>(fn: () => Promise<T>, maxRetries = 3): Promise<T> {
      for (let i = 0; i < maxRetries; i++) {
        try {
          return await fn()
        } catch (error) {
          if (i === maxRetries - 1) throw error
          await new Promise(resolve => setTimeout(resolve, 2 ** i * 1000))
        }
      }
      throw new Error('Max retries exceeded')
    }
    ```
  </Accordion>

  <Accordion title="Batch Requests">
    ```typescript
    // Instead of multiple getProgramAccounts calls
    const agents = await client.agents.getAllAgents({
      filters: { owner: walletAddress }
    })

    // Use batch fetch for multiple accounts
    const accounts = await rpc.getMultipleAccounts([addr1, addr2, addr3]).send()
    ```
  </Accordion>

  <Accordion title="Use Caching">
    ```typescript
    const client = new GhostSpeakClient({
      cluster: 'devnet',
      cache: {
        enabled: true,
        ttl: 60000, // 60 seconds
      },
    })
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting Network Issues

| Issue | Cause | Solution |
|-------|-------|----------|
| **429 Too Many Requests** | Rate limit exceeded | Use paid RPC provider or implement backoff |
| **Connection timeout** | RPC overloaded | Switch to alternative RPC endpoint |
| **Blockhash not found** | Transaction too old | Fetch fresh blockhash before signing |
| **Account not found** | Wrong network | Verify `cluster` configuration |
| **Signature verification failed** | Network mismatch | Ensure signer network matches RPC network |
